02. Project Details

# Project Details

For this project, you will create DApp notary service leveraging the Ethereum platform. You write smart contract that offer securely prove the existence for any digital asset - in this case unique stars and their metadata!

To help you develop such a DApp, we’ve outlined the parts needed below. We will cover these parts in the remainder of this project lesson. There is an optional section if you would like an extra challenge, but note this is not required for the project to pass.

Part 1 Write a smart contract with functions to support proof of existence (i.e. notarization)
Part 2 Test smart contract code coverage
Part 3 Deploy smart contract on a public test network (Rinkeby)
Part 4 Modify client code to interact with a smart contract
Optional Configure RESTful API endpoint to return a registered star

## Expected Challenges

As a note of caution,

  • Solidity is picky! - There will be limitations for the way you work with strings. For instance, if we desire to perform concatenation. To perform this, you will need to create a function to support string manipulation.
  • Uniqueness by coordinates will be challenging due to the limitations with Solidity to iterate over mappings.

If you need some initial guidance, check out the resources we’ve provided for you in the Extra Support - Solidity Review; however, always remember to review the full developer documentation.
